데이터 전처리 이론 — 척도 · 결측값 · 이상값 (비전공자용)

실습 코드는 제외하고 “개념 + 어떤 함수/도구를 쓰는지 + 판단 기준(왜 그렇게 하는지)” 중심으로 정리 · 생성일: 2026-02-09
척도(자료의 종류) 결측값 이상값 실무 체크리스트
0. 데이터 이해

척도의 종류(자료의 종류)

전처리는 “무조건 깨끗하게 만드는 작업”이 아니라, 데이터의 성격에 맞게 다듬는 과정입니다. 그래서 결측값/이상값을 다루기 전에 “이 데이터가 어떤 척도인지”를 먼저 정리합니다.

척도 구분 특징 예시 타이타닉 예시
명목척도
(Nominal)
  • “라벨 붙이기” 수준의 분류만 가능
  • 서열(순서) 없음
  • 숫자로 바꿔도(예: 남=0, 여=1) 크기 의미가 생기는 건 아님
 성별, 지역, 혈액형, 선호 브랜드 성별(male/female), 승선항구(C/Q/S)
순서척도
(Ordinal)
  • 분류 + 서열(순서) 존재
  • 하지만 “1등과 2등의 차이”가 “2등과 3등의 차이”와 같다고 말할 수 없음
  • 그래서 덧셈/뺄셈 같은 연산은 조심
 만족도(1~5), 등급(A/B/C), 학년 선실등급(Pclass: 1,2,3)
구간척도
(Interval)
  • 서열 + 간격(차이) 의미 있음
  • 하지만 절대 0이 없음 (0이 “없음”을 뜻하지 않음)
  • 그래서 “두 배” 같은 비율 비교는 불가
 섭씨온도, IQ 타이타닉엔 대표 예시가 적음(가상의 점수 등으로 예시 가능)
비율척도
(Ratio)
  • 서열 + 간격 + 절대 0 존재
  • 사칙연산 가능(평균/비율/두 배 비교 가능)
 키, 몸무게, 나이, 소득, 수량, 시간 Age, Fare, SibSp, Parch
왜 이 구분이 결측/이상값 처리와 연결될까?
  • 명목/순서(범주형)은 “평균”이 의미 없어서, 결측값을 평균으로 채우면 안 됩니다 → 보통 최빈값(mode)이나 Unknown 같은 “범주”로 처리합니다.
  • 비율/구간(수치형)은 평균/중앙값이 의미가 있으므로, 결측값을 mean / median으로 대체할 수 있습니다(단, 이상값이 있으면 평균은 위험).
  • 이상값 탐지도 수치형에서 주로 하며, 범주형은 “이상범주(희귀범주)” 처리 관점으로 접근합니다.
1. 결측값

결측값 처리 (Missing Values)

결측값이 문제인 이유
  • 모델/통계 함수가 결측값 때문에 계산을 못 하거나, 자동으로 결측이 있는 행을 빼버려서 표본이 줄어듭니다.
  • 특히 결측이 “특정 그룹에 몰려” 있으면(예: 고령층만 소득 미기재) 편향(Bias)이 생깁니다.
결측값을 먼저 ‘진단’하는 함수들
  • df.info(): 컬럼별 결측/데이터타입 빠르게 파악
  • df.isna() / df.isnull(): 결측 여부(True/False)
  • df.isna().sum(): 컬럼별 결측 개수
  • df.isna().sum() / len(df) * 100: 결측 비율(%)

※ 원본 노트에서 강조된 흐름(정보 확인 → 결측 찾기 → 비율 보고 → 삭제/대치)과 동일하게 구성했습니다.

결측 데이터의 종류(MCAR/MAR/NMAR) — “왜 중요한가?”

결측이 생기는 방식에 따라, 단순히 평균으로 채우는 것이 위험할 수도 있습니다.

  • MCAR (완전 무작위 결측): 다른 변수와도 관련이 거의 없는 “우연한 누락”. 이때는 삭제/단순대치가 비교적 안전한 편입니다.
  • MAR (무작위 결측): 결측이 “다른 변수”와 연관. 예: 나이가 많을수록 소득 미기재. 이 경우 단순 평균 대치는 편향을 만들 수 있어, 그룹별 대치나 모델 기반 대치가 더 적절할 수 있습니다.
  • NMAR (비무작위 결측): 결측이 “자기 자신”과 직접 연관. 예: 소득이 낮을수록 소득을 숨김. 평균으로 채우면 현실을 왜곡할 위험이 큽니다(결측 자체가 의미 있는 신호).

왜 결측 비율에 따라 처리 방법이 달라질까?

  • 결측이 적을 때(예: 5% 미만): 삭제해도 정보 손실이 작고, 모델이 배울 데이터가 충분히 남습니다. 또한 단순 대치로 인한 “가짜 패턴”을 만들 필요가 없습니다.
  • 중간 수준(예: 5~30%): 삭제하면 표본이 꽤 줄어 성능/대표성이 나빠질 수 있습니다. 그래서 대치(Imputation)가 일반적입니다.
  • 많은 수준(예: 30~50%): 단순대치(평균/중앙값)만 쓰면, “원래 다양한 값”이 “비슷한 값으로 몰리는” 현상이 생겨 분산이 줄고(데이터가 단조로워짐), 모델이 잘못 학습할 수 있습니다. 이때 KNN/회귀 같은 “다른 변수 정보를 활용한 대치”를 고려합니다.
  • 너무 많은 수준(예: 50% 이상): 해당 컬럼이 “거의 비어있다”는 뜻이라, 대치로 채워도 대부분이 추정값이 됩니다. 즉, 정보보다 노이즈가 커질 수 있어 삭제가 합리적일 수 있습니다(단, 업무적으로 중요한 컬럼이면 예외).

결측값 처리 방법(실습 코드는 제외, “어떤 함수/도구를 쓰는지” 중심)

방법 언제 쓰나 대표 함수/도구(개념용) 주의점
행/열 삭제 결측 비율이 낮고(예: <5%), 삭제해도 데이터가 충분할 때 dropna() (행 기준/열 기준 옵션) 특정 그룹만 삭제되면 편향이 생김(결측 패턴 확인 필요)
단순 대치 결측 비율이 중간이고, 빠르게 기준선을 만들 때 수치형: fillna(mean/median)
범주형: fillna(mode), mode() 		평균 대치는 이상값에 취약(왜곡 가능) → 중앙값 고려
scikit-learn 대치기 파이프라인(모델 학습/예측)에서 일관되게 결측 처리할 때 sklearn.impute.SimpleImputer 학습/검증/테스트에 동일 규칙 적용(데이터 누수 방지)
KNN 대치 다른 변수들과 유사한 패턴이 있을 때(30~50% 구간에서 자주 언급) sklearn.impute.KNNImputer 스케일/거리 개념 영향을 받음(스케일링 고려)
회귀/모델 기반 대치 결측이 중요한 수치형 변수이고, 다른 변수로 충분히 예측 가능할 때 회귀모델, 트리모델 등(개념) 복잡도 증가, 검증 필수(과적합/누수 주의)
결측 자체를 피처로 NMAR처럼 “결측이 의미”일 수 있을 때 isna()로 결측 여부 플래그 생성(개념) 업무적 해석과 함께 사용(왜 결측인지 설명 가능해야 함)
노트북에서 실제로 등장하는(혹은 연계되는) 핵심 함수/도구 키워드

실습 코드는 제외하되, “무슨 도구를 쓰는지”는 강의에 포함해야 한다는 요청을 반영했습니다.

결측 진단/비율: info(), isna(), isnull(), notna()
단순 대치/요약통계: mean(), median(), mode(), fillna(), dropna()
파이프라인 대치: sklearn.impute.SimpleImputer, sklearn.impute.KNNImputer

2. 이상값

이상값 탐지 및 처리 (Outliers)

이상값이 문제인 이유
  • 평균/표준편차를 크게 바꿔서 전체 통계가 왜곡됩니다.
  • 회귀 같은 모델은 이상값 하나에 끌려가 예측선이 휘어질 수 있습니다.
  • 하지만 이상값이 “오류”가 아니라 “중요한 사건(부정거래/고액결제)”일 수도 있어, 무조건 제거는 위험합니다.
먼저 해야 할 것: 시각화/분포 확인
  • describe()로 최소/최대/사분위수 확인
  • boxplot()으로 수염 밖 점(후보) 확인
  • 분포가 오른쪽으로 긴지(왜도), 꼬리가 두꺼운지(극단값 많음) 확인

이상값 탐지 방법 3가지 (노트북 구조 유지 + 이해 설명 보강)

방법 핵심 아이디어 언제 유리한가 주의점
Z-score
(표준화 점수)		 평균에서 표준편차 기준으로 얼마나 떨어졌는지: (x-평균)/표준편차 정규분포에 가깝고 “극단값”을 빠르게 찾을 때 표준편차 자체가 이상값의 영향을 받으면 탐지가 약해질 수 있음
평균 ± 3σ
(ESD 계열)		 평균을 기준으로 ±(3*표준편차) 밖이면 후보 설명하기 쉽고, 초보자에게 직관적 분포가 비정규/왜도가 크면 과탐/미탐 가능
IQR
(사분위수 범위)		 중앙 50% 구간(IQR)을 기준으로 상/하한을 정함 정규/비정규 모두 비교적 강함(노트북에서도 “강함” 강조) 데이터 특성에 따라 1.5배 기준 조정이 필요할 수 있음

IQR을 비전공자 관점으로 풀어 설명

상자수염그림(Boxplot)에서 가운데 상자(박스)가 “중앙 50%”입니다. 이 상자의 길이가 IQR이고, 보통 Q1 - 1.5×IQR보다 작거나 Q3 + 1.5×IQR보다 큰 점을 “이상값 후보”로 봅니다.

왜 평균/표준편차보다 안전하냐면, 평균은 극단값 하나에 흔들리지만, 사분위수(Q1/Q3)는 “순서 기반”이라 극단값 영향이 상대적으로 작기 때문입니다.

왜 ‘이상값 처리’도 상황에 따라 방법이 달라질까?

  • 오류(out-of-range, 입력 실수)면 수정/삭제가 맞습니다. 예: 나이 999.
  • 희귀하지만 가능한 값이면, 삭제하면 “현실의 극단”을 못 배우게 됩니다. 예: 매우 높은 운임(Fare)도 실제 VIP가 있을 수 있음.
  • 목표가 무엇인지에 따라 다릅니다. 예: “일반 사용자 평균 행동”을 모델링하면 극단값을 줄이는 게 유리할 수 있고, “부정거래 탐지”면 극단값이 핵심 신호입니다.

이상치 처리 방법 6가지 (노트북 표 유지 + 도구/함수 힌트 추가)

방법 설명 대표 도구/함수(개념용) 언제 추천
1. 삭제 이상치로 판단되는 행(row)을 제거 drop(), 조건 필터링(개념) 데이터가 충분히 많고, 이상치가 “오류”로 확실할 때
2. 대체 평균/중앙값 등으로 값을 수정 median(), fillna(), 조건 치환(개념) 극단값이 소수이고, 분포를 크게 바꾸고 싶지 않을 때
3. 변환 로그/루트 변환으로 “큰 값의 영향”을 완화 log, sqrt(개념) 오른쪽으로 긴 분포(우측 꼬리)에서 자주 사용
4. 구간 압축
(Winsorization)		 상/하위 극단값을 상/하한 값으로 “컷”하여 영향만 줄임 clip(), scipy.stats.mstats.winsorize() 삭제는 싫지만, 평균 왜곡은 줄이고 싶을 때
5. Robust 방법 사용 이상치에 덜 민감한 스케일링/모델 사용 RobustScaler, RANSAC(개념) 이상치가 섞여 있어도 모델이 견딜 필요가 있을 때
6. 별도 분석 이상치만 따로 분석해서 인사이트(이벤트/사기/특수고객) 도출 세그먼트 분석(개념) 이상치가 “중요한 의미”일 수 있는 도메인(금융/보안 등)
노트북에서 실제로 등장하는(혹은 연계되는) 핵심 함수/도구 키워드

탐지: mean(), std(), quantile(), describe(), boxplot(), scipy.stats.zscore()
처리: clip(), scipy.stats.mstats.winsorize(), (분포 완화) log/sqrt 변환

3. 실무 체크리스트

결측값/이상값 처리 체크리스트

  1. 척도 확인: 범주형인지 수치형인지부터 분리(명목/순서/구간/비율).
  2. 결측 패턴 확인: “어느 컬럼/어느 그룹에” 결측이 몰렸는지.
  3. 결측 비율 기준 적용: <5%는 삭제 가능, 5~30%는 단순대치, 30~50%는 KNN/모델대치 고려, ≥50%는 컬럼 중요도 평가 후 삭제/대치 결정.
  4. 이상치 후보 탐지: Z-score/3σ/IQR 중 데이터 분포에 맞는 기준 선택.
  5. 이상치의 의미 판단: 오류인지(삭제/수정), 희귀한 정상인지(변환/윈저라이즈/Robust), 의미 있는 이벤트인지(별도 분석).
  6. 설명 가능성: “왜 삭제/대치/압축했는지”를 문장으로 설명할 수 있어야 함(특히 금융/공공).

※ 실습 코드는 제외했지만, 강의에서는 위 체크리스트를 “실제 데이터(타이타닉 등)”에 대입해 판단 연습을 시키면 효과가 좋습니다.
