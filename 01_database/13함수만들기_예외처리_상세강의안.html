1. 함수란 무엇인가

함수(Function)는 여러 줄의 코드를 하나의 이름으로 묶어 필요할 때마다 호출해 사용하는 구조입니다.

  • 이미 써 본 함수: print(), input(), len(), sum(), str.split() 등
  • 함수를 만드는 이유
  • 중복 제거: 같은 코드를 반복해서 쓰지 않음
  • 가독성: 함수 이름만 봐도 역할을 알 수 있음
  • 재사용: 한 번 만들어 여러 곳에서 사용
  • 테스트/디버깅: 기능 단위로 점검 가능

실습 1: 중복 코드 확인

print("안녕하세요")
print("안녕하세요")
print("안녕하세요")

2. 함수의 기본 구조

함수는 def로 정의하고, ()로 호출합니다.

def 함수이름():
    실행할 코드

함수이름()
  • def는 define(정의)
  • : 다음은 들여쓰기(블록)
  • 정의만으로는 실행되지 않음 → 반드시 호출해야 실행됨

실습 2: 가장 단순한 함수

def say_hello():
    print("안녕하세요")

say_hello()

파이썬 코드는 기본적으로 위에서 아래로 실행됩니다. 따라서 함수는 호출되기 전에 정의되어 있어야 합니다.

# 실행하면 오류가 납니다(함수 정의 전 호출)
# say_hi()

def say_hi():

say_hi()

실습 3-2: 올바른 예

4. 함수의 매개변수(Parameter)

매개변수는 함수가 외부에서 값을 받아 처리하도록 만드는 장치입니다.

  • 매개변수(parameter): 함수 정의에서 받는 변수
  • 인수(argument): 함수 호출 때 전달하는 값

실습 4: 매개변수 1개 함수

def greet(name):
    print(f"{name}님, 환영합니다!")

greet("철수")
greet("영희")

5. 인수(Argument) 전달 방식 총정리

이 단원에서는 아래 개념을 한 번에 정리합니다.

  • 위치 인수(positional argument)
  • 키워드 인수(keyword argument)
  • 가변 위치 인수(*args)
  • 가변 키워드 인수(**kwargs)
  • 기본값(초기값) 지정 시 매개변수 순서 규칙

5-1. 위치 인수 (positional argument)

  • 위치(순서)가 고정된 인수
  • 순서가 바뀌면 다른 매개변수에 들어갑니다.
  • 개수가 맞지 않으면 TypeError가 발생합니다.

실습 5-1

def minus(a, b):
    return a - b

print(minus(10, 3))   # a=10, b=3
print(minus(3, 10))   # a=3,  b=10  (결과가 달라짐)

# print(minus(10))    # TypeError: 인수 부족
# print(minus(10, 3, 1))  # TypeError: 인수 과다

5-2. 키워드 인수 (keyword arguments)

  • 매개변수 이름을 명시해서 값을 넣는 방식
  • 순서를 바꿔도 이름이 같으면 올바르게 들어갑니다.
  • 가독성이 좋아지고, 실무에서 실수 방지에 도움이 됩니다.

실습 5-2

def profile(name, age, city):
    return f"{name} / {age} / {city}"

print(profile("철수", 20, "서울"))                # 위치 인수
print(profile(age=20, name="철수", city="서울"))  # 키워드 인수 (순서 바뀌어도 OK)

5-3. 가변 위치 인수 *args (Variable-length Arguments)

  • 위치 인수를 여러 개 받을 때 사용합니다.
  • 개수 제한이 없으며, 함수 안에서는 tuple로 들어옵니다.

실습 5-3

def sum_all(*args):
    # args는 튜플
    total = 0
    for x in args:
        total += x
    return total

print(sum_all(1, 2, 3))
print(sum_all(10, 20, 30, 40, 50))
print(sum_all())  # 0

5-4. 가변 키워드 인수 **kwargs (variable-length keyword arguments)

  • 키워드 인수를 여러 개 받을 때 사용합니다.
  • 개수 제한이 없으며, 함수 안에서는 dict로 들어옵니다.

실습 5-4

def show_kwargs(**kwargs):
    # kwargs는 딕셔너리
    for k, v in kwargs.items():
        print(k, "=", v)

show_kwargs(name="철수", age=20, city="서울")

5-5. (중요) 기본값(초기값) 지정 시 매개변수 순서 규칙

사용자가 요청한 핵심 규칙:

✅ 함수 정의에서 매개변수는 보통 아래 순서를 따른다.

1) 위치(일반) 매개변수 2) 기본값(초기값)이 있는 매개변수 *(키워드 인수로도 호출 가능)* 3) **가변 위치 인수 *args 4) 가변 키워드 인수 kwargs

이 순서를 지키지 않으면 SyntaxError가 발생할 수 있습니다.

실습 5-5: 올바른 순서 예시

def demo(a, b, c=0, d=0, *args, **kwargs):
    print("a, b =", a, b)
    print("c, d =", c, d)
    print("args =", args)
    print("kwargs =", kwargs)

demo(1, 2)
demo(1, 2, 3, 4, 5, 6, x=10, y=20)

실습 5-5(참고): 잘못된 순서(주석 해제하면 SyntaxError)

# def wrong(a=1, b):   # 기본값 있는 매개변수가 앞에 오면 안 됨
#     ...

6. 함수의 반환값(return)

return은 함수의 결과를 함수 밖으로 돌려주는 것입니다.

  • print()는 화면에 보여주기
  • return은 값을 돌려주기 → 다른 계산에 재사용 가능

실습 6-1: return이 없는 함수의 특징

def show_sum(a, b):
    print(a + b)

x = show_sum(3, 4)
print("show_sum의 반환값:", x)  # None

실습 6-2: return이 있는 함수

def add(a, b):
    return a + b

result = add(10, 20)
print(result)
print(result * 2)

실습 6-3: 실무 느낌 예제(세금 계산)

def calc_tax(price, tax_rate):
    return price * tax_rate

vat = calc_tax(10000, 0.1)
print(vat)

함수 안에서 만든 변수는 지역 변수(Local Variable) 입니다.

  • 함수 안에서만 사용 가능
  • 함수 밖에서는 접근 불가

실습 7: 지역 변수 범위 확인

def test():

9. 조건문과 함께 사용하는 함수

조건문을 함수에 넣으면 입력값에 따라 다른 처리를 할 수 있습니다.

실습 9-1: 짝수/홀수 판별

def check_even(num):

        print("짝수")

def grade(score):
    if score >= 90:
        return "A"
    elif score >= 80:
        return "B"
    elif score >= 70:
        return "C"
    else:
        return "D"

print(grade(85))

11. 함수 docstring 사용하기

11-1. docstring이란?

docstring(독스트링)은 함수의 기능/사용법을 함수 내부에 문서로 남기는 문자열입니다.

def 함수이름(매개변수):
    """독스트링으로 설명"""
  • docstring은 함수를 실행할 때 자동 출력되지 않습니다.
  • 대신 아래 방법으로 확인합니다.
  • print(함수이름.__doc__)
  • help(함수이름)
참고: __doc__처럼 언더바 2개가 정확한 속성 이름입니다. (_doc_가 아님)

실습 11: docstring 작성/확인

def add(a, b):
    """두 수 a, b를 더해서 결과를 반환합니다.

    사용 예:
        add(3, 4) -> 7
    """
    return a + b

print(add(3, 4))
print(add.__doc__)
help(add)  # 주피터에서는 출력이 길 수 있어요

12. 함수 활용 패턴 정리

  • 계산용 함수: return으로 결과를 돌려줘 재사용
  • 데이터 처리용 함수: 리스트/딕셔너리 등을 가공

실습 12-1: 평균 계산(리스트 처리 + 계산)

def mean(nums):
    return sum(nums) / len(nums)

print(mean([10, 20, 30]))

13. 람다 표현식(lambda expression)

13-1. lambda란?

lambda는 이름이 없는(익명) 함수를 빠르게 만드는 문법입니다.

  • 간단한 함수를 짧게 작성할 때
  • 다른 함수의 인수로 “함수”를 넘길 때(예: sorted, map 등)
  • 보통 일회성으로 사용(사용 후 참조가 없으면 메모리에서 해제)

기본 형태:

lambda 매개변수: 반환할 값(표현식)

13-2. 실습 13-1: 즉시 실행

print((lambda x: x + 10)(5))

실습 13-2: 변수에 저장해서 사용

plus_five = lambda x: x + 5
print(plus_five(5))

실습 13-3: 다른 함수의 인수로 사용(sorted의 key)

data = [("철수", 90), ("영희", 70), ("민수", 85)]
# 점수 기준으로 정렬
sorted_data = sorted(data, key=lambda x: x[1])
print(sorted_data)

16. 예외 처리(Exception Handling)

예외 처리는 오류가 발생해도 프로그램이 멈추지 않도록 만드는 방법입니다.

기본 구조:

try:
    위험한 코드
except:
    오류가 나면 실행할 코드

실습 16: try-except 기본

try:
    num = int(input("숫자 입력: "))
    print("10 / num =", 10 / num)
except:
    print("오류 발생: 입력값을 확인하세요")

17. 여러 개의 예외 처리하기

모든 오류를 한 번에 잡는 것보다, 오류 종류별로 정확히 처리하는 것이 좋습니다.

실습 17: 오류 종류별 처리

try:
    num = int(input("숫자 입력: "))
    print("10 / num =", 10 / num)
except ValueError:
    print("숫자를 입력하세요(정수로 변환 실패)")
except ZeroDivisionError:
    print("0은 나눌 수 없습니다")

18. else / finally

  • else: try가 성공했을 때만 실행
  • finally: 오류가 나든 말든 항상 실행

실습 18: else/finally 확인

try:
    num = int(input("숫자: "))
except ValueError:
    print("숫자가 아닙니다")
else:
    print("정상 입력:", num)
finally:
    print("프로그램 종료(정리 작업 위치)")

19. 함수 + 예외 처리 결합(실무형 패턴)

실무에서는 사용자 입력이 항상 정상이라고 가정하면 안 됩니다. 따라서 보통 함수 안에서 입력을 받고, 예외 처리로 안전하게 만듭니다.

실습 19-1: 안전한 나눗셈 함수

def safe_divide():
    try:
        a = int(input("a: "))
        b = int(input("b: "))
        print("a / b =", a / b)
    except ValueError:
        print("정수로 입력해야 합니다")
    except ZeroDivisionError:
        print("0으로 나눌 수 없습니다")

실습 19-2(확장): 재입력 루프 패턴

def safe_input_int(prompt):
    while True:
        try:
            return int(input(prompt))
        except ValueError:
            print("정수로 다시 입력해주세요.")

20. 전체 내용 정리

  • 함수는 코드를 재사용/구조화하는 도구
  • 인수 전달 방식(위치/키워드/가변)은 실수 방지와 확장성에 중요
  • return은 결과를 값으로 돌려줘 재활용 가능
  • docstring은 함수의 내장 설명서
  • lambda는 짧은 익명 함수로, 다른 함수에 전달할 때 유용
  • 예외 처리는 프로그램 안정성을 높임
  • 입력 검증 + 예외 처리는 실무에서 거의 필수

  • op는 "+", "-", "*", "/" 중 하나
  • 숫자가 아닌 값 입력 시 ValueError 처리
  • 나눗셈에서 0 입력 시 ZeroDivisionError 처리
  • 정상 계산이면 결과 출력, 오류면 이유 출력

힌트: 함수 내부에서 try-except를 사용하고, if/elif로 op를 분기한다.

def calc(op):
    try:
        a = int(input("첫 번째 정수: "))
        b = int(input("두 번째 정수: "))

        if op == "+":
            print(a + b)
        elif op == "-":
            print(a - b)
        elif op == "*":
            print(a * b)
        elif op == "/":
            print(a / b)  # b가 0이면 아래 except에서 처리
        else:
            print('지원하지 않는 연산자입니다. (+, -, *, / 중 하나)')
    except ValueError:
        print("정수만 입력해야 합니다.")
    except ZeroDivisionError:
        print("0으로 나눌 수 없습니다.")
