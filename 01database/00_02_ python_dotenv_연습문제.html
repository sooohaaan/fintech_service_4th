python-dotenv 연습문제

본 연습문제는 정리된 강의안(설치 · .env 작성 · 기본 사용 · 경로/이름 변경 · 미니 실습 · .gitignore)을 기준으로 구성되었다.
힌트는 방향만 제시하며, 정답은 별도로 구분되어 있다.

세트 A. 설치 & 기본 개념

1번 python-dotenv 설치

python-dotenv 패키지를 설치하고, 정상적으로 import 되는지 확인하시오.

힌트

pip를 이용해 설치한다.

정답 
pip install python-dotenv

from dotenv import load_dotenv

2번 .env 파일 작성

.env 파일을 만들고 아래 설정을 작성하시오.

  • APP_ENV=dev
  • DEBUG=true
  • GREETING="Hello dotenv"
힌트

KEY=VALUE 형식이며, 문자열에 공백이 있으면 따옴표를 사용한다.

정답 
APP_ENV=dev
DEBUG=true
GREETING="Hello dotenv"

세트 B. 기본 사용법

3번 load_dotenv 기본 사용

.env 파일을 로딩하고 GREETING 값을 출력하시오.

힌트

load_dotenv() 호출 후 os.getenv()를 사용한다.

정답 
from dotenv import load_dotenv
import os

load_dotenv()
print(os.getenv("GREETING"))

4번 기본값 처리

TIMEOUT 환경변수가 없을 때 기본값 30을 사용하도록 작성하시오.

힌트

os.getenv의 두 번째 인자를 사용한다.

정답 
timeout = int(os.getenv("TIMEOUT", "30"))

세트 C. 경로 · 이름이 다른 .env

5번 파일 이름이 다른 경우

.env.dev 파일을 로딩하도록 코드를 작성하시오.

힌트

load_dotenv 함수에 파일명을 직접 전달한다.

정답 
from dotenv import load_dotenv

load_dotenv(".env.dev")

6번 다른 경로에 있는 .env

프로젝트 외부 경로(/etc/myapp/secrets.env)에 있는 env 파일을 로딩하시오.

힌트

Path를 이용해 절대경로를 지정한다.

정답 
from dotenv import load_dotenv
from pathlib import Path

load_dotenv(Path("/etc/myapp/secrets.env"))

7번 실행 위치와 무관하게 로딩

실행 위치에 상관없이 항상 같은 .env를 읽도록 작성하시오.

힌트

__file__ 기준 경로를 사용한다.

정답 
from pathlib import Path
from dotenv import load_dotenv

BASE_DIR = Path(__file__).resolve().parent
load_dotenv(BASE_DIR / ".env")

세트 D. 미니 실습 (Jupyter Notebook)

8번 ipynb 실습

Jupyter Notebook 셀에서 .env를 로딩하고 설정 값을 출력하시오.

  • APP_ENV
  • DEBUG
  • GREETING
힌트

함수 정의 없이 셀에서 바로 실행한다.

정답 
from dotenv import load_dotenv
import os

load_dotenv()

print(os.getenv("APP_ENV"))
print(os.getenv("DEBUG"))
print(os.getenv("GREETING"))

9번 .gitignore 설정

.env 파일이 Git에 올라가지 않도록 .gitignore를 작성하시오.

힌트

모든 .env 계열은 제외하고, 예제 파일만 허용한다.

정답
