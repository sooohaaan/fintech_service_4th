01. HTML 기초 — 정적 수집의 출발점

크롤링은 결국 “HTML(또는 렌더링된 결과)에서 원하는 요소를 찾아 텍스트/속성 값을 추출”하는 작업이다. 이 문서는 poiemaweb의 HTML5 문법/요소/속성/시맨틱/폼 핵심을 수집 관점으로 압축했다.

학습 목표 HTML5 문서 구조/문법 요소(Element)와 중첩 빈 요소(Empty element) 속성(Attribute) & 글로벌 속성 시맨틱 요소(SEO/크롤링) data-* (수집에 특히 유용) 폼(Form)과 input (데이터 전송) 정적 vs 동적 미니 실습 링크/이미지(속성 추출) 목록/반복 구조(카드/리스트) 표(table) 구조(행/열) 오디오/비디오(미디어 태그) 추가 실습(수집용 HTML 만들기)

학습 목표

  • HTML5 문서가 어떤 구조로 구성되는지(doctype/head/body) 이해한다.
  • 요소/중첩/빈 요소/속성(id, class 등)을 “수집 관점”에서 해석할 수 있다.
  • 시맨틱 요소가 SEO/크롤링에 왜 도움이 되는지 안다.
  • 폼과 input이 어떤 방식으로 데이터를 서버로 보내는지 감을 잡는다.
poiemaweb 기반 핵심
HTML5 문서는 <!DOCTYPE html>로 시작하고, 실제 화면 요소는 <body>에 위치하며, 요소는 중첩되고 속성으로 특징을 가진다.

HTML5 문서 구조/문법

HTML5 문서의 기본 틀은 아래처럼 반복된다. (head는 “설정/메타”, body는 “화면 콘텐츠”) 

<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8">
    <title>Hello World</title>
  </head>
  <body>
    <h1>Hello World</h1>
    <p>안녕하세요! HTML5</p>
  </body>
</html>
크롤링 관점
  • 정적 수집은 보통 “서버가 준 HTML 문자열”을 파싱한다.
  • 동적 사이트는 <body>가 비어있다가 JS로 채워질 수 있다.
이 구조/설명은 poiemaweb의 HTML5 Syntax(doctype/head/body, 요소/속성/중첩) 내용을 기반으로 한다.

요소(Element)와 중첩(Nested)

HTML은 “요소들의 집합”이다. 요소는 보통 시작 태그 + 종료 태그 + content로 이루어지고, 다른 요소를 포함(중첩)할 수 있다. 

<div class="card">
  <h2>제목</h2>
  <p>설명</p>
</div>
수집 팁
중첩 구조 덕분에 “상위 컨테이너(카드/행/리스트)”를 잡고 그 안에서 필요한 값(제목/가격)을 찾는 방식이 안정적이다.
요소/중첩 개념은 poiemaweb HTML5 Syntax의 “요소/중첩” 파트를 따른다.

빈 요소(Empty element)

content를 가질 수 없는 요소를 “빈 요소”라고 한다. 대표적으로 img, input, meta, link 등이 있다. 

<meta charset="utf-8">
<img src="/img/a.png" alt="썸네일">
<input type="text" name="q">
크롤링 관점
  • 빈 요소는 텍스트가 아니라 “속성 값(src, alt, name 등)”을 주로 뽑는다.
빈 요소/예시는 poiemaweb HTML5 Syntax의 “Empty element” 설명과 일치한다.

링크/이미지(속성 추출)

크롤링에서 링크와 이미지는 “텍스트”보다 속성 값을 추출하는 경우가 많다. 특히 href, src, alt는 자주 쓰인다. 

<a href="/product/123" class="item-link">상품 보기</a>
<img src="/img/123.png" alt="상품 썸네일">
크롤링 포인트
  • 상대 경로(/product/123)면 base URL과 합쳐 절대 경로로 만든다.
  • img는 빈 요소라서 텍스트가 아니라 src/alt를 뽑는다.
  • 썸네일이 data-src에 있거나 lazy-loading이면 data-*도 확인한다.
링크/이미지/속성(alt, src, href) 개념은 기본 HTML 요소/속성 설명 흐름과 동일하다.

목록/반복 구조(카드/리스트)

쇼핑몰/게시판/랭킹 같은 페이지는 보통 “반복되는 목록” 구조다. HTML에서는 ul > li, ol > li 형태가 자주 나오고, 실무 페이지는 div 기반 “카드(card)” 반복도 많다.

1) ul/ol 목록

<ul class="list">
  <li class="item">...</li>
  <li class="item">...</li>
</ul>

2) 카드 반복

<div class="cards">
  <div class="card item">...</div>
  <div class="card item">...</div>
</div>
수집 전략
  • 먼저 “반복 컨테이너(목록)”를 잡고 → 그 안에서 제목/가격/링크를 찾는다.
  • 반복 구조를 잡는 선택자는 보통 .item, .card, li 같은 단위가 된다.

표(table) 구조(행/열)

데이터가 “행/열”로 정리된 페이지는 table을 사용한다. 특히 공공데이터, 통계, 요금표, 일정표에서 자주 보인다. 

<table>
  <thead>
    <tr><th>이름</th><th>가격</th></tr>
  </thead>
  <tbody>
    <tr><td>A</td><td>1000</td></tr>
  </tbody>
</table>
크롤링 포인트
  • tr 하나가 “한 행(row)”이다. 보통 tbody tr을 반복 수집한다.
  • 제목 행이 thead에 있거나 첫 tr에 섞여 있을 수 있다.
  • 셀은 th(헤더), td(데이터)로 구분된다.

오디오/비디오(미디어 태그)

audio, video는 미디어를 재생하는 태그다. 수집 관점에서는 “미디어 파일 URL”(src)을 찾는 케이스가 많다. 

<audio src="/media/a.mp3" controls></audio>
<video src="/media/v.mp4" controls poster="/img/thumb.jpg"></video>
크롤링 포인트
  • src가 비어 있고 <source> 태그로 제공되는 경우도 있다.
  • 플레이어가 커스텀이면(태그가 아니라 div) 실제 URL은 Network에서 찾는 편이 빠르다.

속성(Attribute) & 글로벌 속성

속성은 요소의 성질/특징을 정의하며, 보통 name="value" 형태다. 

<img src="html.jpg" width="104" height="142">
글로벌 속성 의미 크롤링에서의 활용
id 유일한 식별자(중복 불가) 안정적이면 최우선 선택자 후보
class 그룹 라벨(중복 가능) 리스트/카드 반복 요소를 잡는 데 자주 사용
title 툴팁/설명 가끔 실제 데이터(전체 제목)가 들어있다
lang 요소 언어 검색엔진/크롤러가 언어를 인지하는 데 도움
글로벌 속성(id/class/lang/title 등) 설명은 poiemaweb HTML5 Syntax의 글로벌 어트리뷰트 내용을 반영했다.

시맨틱 요소(SEO/크롤링)

시맨틱 요소는 “콘텐츠의 의미”를 태그 이름 자체로 전달한다. 예: header, nav, main, section, article, footer. 이런 구조는 검색엔진이 문서를 더 의미 있게 해석(크롤링/인덱싱)하는 데 도움이 될 수 있다. 

<header>...</header>
<main>
  <article>게시글</article>
</main>
<footer>...</footer>
시맨틱 웹/시맨틱 태그가 크롤링과 SEO에 유리할 수 있다는 관점은 poiemaweb 시맨틱 웹 문서의 취지(검색엔진이 문서 의미를 더 잘 파악)와 맞닿아 있다.

data-* (수집에 특히 유용)

실무에서 개발자는 화면 표시용 텍스트(“12,900원”)와 별개로, 정제된 값을 data-* 속성에 넣어두는 경우가 많다. 

<button class="buy"
  data-sku="VAC-001"
  data-price="129000">
  구매
</button>
크롤링 포인트
  • 화면 텍스트 파싱(쉼표/원 제거) 대신, data-price 같은 “원본 숫자”를 쓰면 오류가 줄어든다.

폼(Form)과 input — 데이터 전송 방식 이해

폼은 사용자가 입력한 데이터를 서버로 전송하는 표준 방식이다. method와 action, 그리고 input의 name=value 형태가 핵심이다. 

<form action="/search" method="get">
  <input type="text" name="q" value="vacuum">
  <input type="submit" value="Search">
</form>
수집 관점
  • 검색 기능이 “폼 GET”이면 URL 쿼리(?q=...)가 만들어진다 → 크롤링에서 재현하기 쉽다.
  • form의 name이 서버가 받는 키가 되므로, 입력 파라미터를 이해하는 데 중요하다.
form/action/method, input의 name=value 전송 개념은 poiemaweb의 HTML5 Forms 문서 핵심 흐름을 반영했다.

정적 vs 동적 (개념만)

구분 정적(Static) 동적(Dynamic)
데이터 위치 처음 받은 HTML에 있음 JS 실행 후 DOM에 채워짐 / Network에서 JSON으로 옴
첫 전략 HTML 파싱 + Selector Network에서 API 찾기 또는 브라우저 자동화
동적 페이지는 DOM을 JS로 변경할 수 있고, 변경된 DOM이 렌더링에 반영된다는 관점은 poiemaweb DOM 문서의 설명과 일치한다.

미니 실습

  1. 아무 웹페이지에서 F12 → Elements 열기
  2. 제목/가격/작성일 같은 텍스트를 클릭해서 HTML 위치 확인
  3. 해당 요소의 id/class/data-*가 있는지 체크
  4. 검색창이 있다면 input의 name을 확인(폼 파라미터 추정)
힌트
  • “반복되는 카드/행”을 먼저 찾으면 selector 설계가 쉬워진다.
  • 폼의 method가 GET이면 주소창 URL에 파라미터가 보이는 경우가 많다.

추가 실습(수집용 HTML 만들기)

크롤링 연습을 위해 “내가 만든 HTML”을 준비해두면 selector 연습이 빨라진다. 아래 요구사항대로 practice.html을 직접 만들어보자.

요구사항
  1. 상품 카드 5개를 만든다. (제목, 가격, 링크, 썸네일)
  2. 각 카드는 div class="item"로 통일한다.
  3. 가격은 화면 텍스트(“12,900원”)와 data-price="12900"를 함께 넣는다.
  4. 랭킹 목록(ol)을 만들고 상위 3개만 선택할 수 있게 구조를 만든다.
  5. 표(table)로 “배송비 안내”를 작성한다.
힌트(예시 뼈대) 
<div class="item" data-price="12900">
  <a class="item-link" href="/p/1">무선 청소기</a>
  <span class="price">12,900원</span>
</div>
